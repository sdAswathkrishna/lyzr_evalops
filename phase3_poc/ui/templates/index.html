The Gap

What Lyzr has vs. what's missing

✓ Lyzr Already Has

  • • Agent tracing with spans, logs, tokens, cost, latency
  • • Agent versions and saved configurations
  • • Studio-side simulation and monitoring
  • • Workflow routing and orchestration
  • • Trace APIs for some agent types
  • • Responsible AI guardrails and policies

✗ What's Missing

  • • Code-first workflow to convert traces into eval datasets
  • • Programmable LLM-as-a-judge evaluator framework
  • • Experiment comparison with metric deltas
  • • Automated release decisions (promote/canary/hold/rollback)
  • • Unified simulation → production → eval loop
  • • Routing quality analytics (misroute rate, fallbacks)

Key Insight: Lyzr's gap is not "no evaluation capability" — it's missing the code-first EvalOps loop that connects tracing, dataset creation, evaluator execution, version comparison, and automated release decisions in one workflow.
